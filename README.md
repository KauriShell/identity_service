# identity_service

## Docker setup

1. Copy environment template:
   `cp .env.example .env` (defaults in `.env` are already set for local Docker)

2. Build and start services (from the repo root):
   `docker compose up --build`

3. Run migrations if needed:
   `docker compose exec app ./bin/rails db:prepare`

4. Run the test suite:
   `docker compose exec app bundle exec rspec`

The app listens on `http://localhost:3000`. PostgreSQL and Redis are only exposed to the Docker network.
Swagger UI is available at `http://localhost:3000/api-docs`. The default `swagger/v1/swagger.yaml` is included and can be regenerated by running `bundle exec rspec spec/requests/api_docs`.

Solid Queue / Jobs dashboard (Mission Control) is available at `http://localhost:3000/jobs`.
It is HTTP basic-auth protected. Use:
- username: `dev` (override with `MISSION_CONTROL_JOBS_USER`)
- password: `change-me` (override with `MISSION_CONTROL_JOBS_PASSWORD`)
When running via Docker Compose, these env vars are set automatically in `docker-compose.yml`.

TrustBridge (Next.js) landing app is available at `http://localhost:3001/` (repo root `docker compose` maps `3001` → container `3000`).

## Local (non-Docker)

Set the required environment variables (see `.env.example`), run `bundle install`,
then `bin/rails db:prepare` and `bin/rails server`.
